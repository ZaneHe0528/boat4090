# 模型评估指南（警戒线中线与航行可用性）

## 1. 目标
本指南用于回答两个问题：
1. 分割模型是否在验证集上表现稳定。
2. 模型是否已经能可靠计算警戒线中线，并支持小船沿中线行驶。

## 2. 评估历史与当前结论

| 训练轮次 | loss 函数 | boundary IoU | boundary recall | 状态 |
|---|---|---|---|---|
| 100 epoch（`class_weights=[0.5,1,2]`）| CrossEntropy | 0.0006 | ~0 | ❌ 不可用 |
| 100 epoch（`class_weights=[1,2,20]`）| CrossEntropy | 0.0887 | 0.2169 | ❌ 不达标 |
| 120 epoch（`class_weights=[1,2,20]` + Dice+Focal + 膨胀 + 过采样）| DiceFocal | **训练中** | — | ⏳ 训练中 |

**当前结论**：单纯提高 loss 权重效果有限；正在使用 Dice+Focal 组合损失 + 标注膨胀 + 边界感知采样三项改进重新训练，训练完成后重新评估。

## 3. 一键评估命令
在项目根目录执行：

```bash
python training/evaluate.py
```

默认会读取：
- 模型：`models/segformer_river/best_model.pth`
- 验证图像：`dataset_final/images/val`
- 验证标签：`dataset_final/masks/val`

输出包括：
- mIoU 与各类别 IoU
- 分类报告（precision/recall/F1）
- 中线指标（MAE/P95/Coverage）
- 混淆矩阵图：`models/segformer_river/confusion_matrix.png`

## 4. 历次实际评估结果

### 第一次（CrossEntropy, weights=[0.5,1,2]）
- `mIoU = 0.6062` | `IoU(boundary) = 0.0006` | `boundary recall ≈ 0`

### 第二次（CrossEntropy, weights=[1,2,20]）
- `mIoU = 0.6250` | `IoU(boundary) = 0.0887` | `boundary recall = 0.2169`
- `Centerline MAE = 10.28 px`（`norm = 0.0161`）| `Centerline P95 = 64.05 px`
- 问题：precision=0.13，boundary 仍大量漏检或误判，loss 权重提升效果已接近瓶颈

### 第三次（DiceFocal, 膨胀+过采样）
- 训练中，完成后执行 `python training/evaluate.py` 获取结果

## 5. 如何判断“能否准确算中线并让船沿中线行驶”
建议同时满足以下门槛：

1. 分割安全性门槛（必须）
- `IoU(boundary) >= 0.60`（最低可试）
- `recall(boundary) >= 0.70`

2. 中线精度门槛（必须）
- `Centerline MAE(norm) <= 0.03`
- `Centerline P95 <= 0.08 * 图像宽度`
- `Centerline Coverage >= 0.95`

3. 时序稳定性门槛（强烈建议）
- 相邻帧中线抖动标准差建议 `<= 0.02 * 图像宽度`
- 连续异常跳变帧比例尽量接近 0

4. 闭环航行门槛（实船前必须）
- 平均横向误差（CTE）稳定在可控范围
- 95 分位 CTE 低于安全阈值
- 越界率接近 0

## 6. 为什么当前结果还不够
- `water` 高并不代表可安全导航。
- 导航中线的安全性强依赖边界（警戒线）识别。
- 当前 `boundary` 指标过低，说明模型可能把边界忽略或混入水域，存在越界风险。

## 7. 已采用的改进措施

### ✅ 第一轮：提升 loss 权重
- `class_weights` 从 `[0.5, 1, 2]` 调整为 `[1.0, 2.0, 20.0]`
- 效果：boundary IoU 0.0006 → 0.0887，但接近上限

### ✅ 第二轮（当前正在训练）：三管齐下

**改进 1：Dice + Focal 组合损失函数**
- CrossEntropy 对像素绝对数量求平均，boundary 1.78% 的像素即便权重 20× 也会被压制
- Dice Loss 直接优化 IoU，对小类别天然平衡；Focal Loss 聚焦难预测像素
- 代码位置：`train_segformer.py` → `DiceFocalLoss` 类

**改进 2：boundary 标注膨胀（`boundary_dilate_px=3`）**
- SegFormer 内部有 1/4 降采样，1-3 px 的细线边界经插值后几乎归零
- 将 boundary 标注由细线膨胀为半径 3 px 的粗带，使梯度信号可达
- 代码位置：`RiverDataset.__getitem__` → cv2.dilate

**改进 3：边界感知过采样（WeightedRandomSampler）**
- 158 张训练图中，部分图几乎无 boundary 像素；靠随机采样低效学习
- 按每张图的 boundary 像素占比加权，boundary 丰富的图被更频繁采样
- 代码位置：`Trainer._compute_sample_weights`

### 后续如仍不达标
1. 增加数据量：补充更多含边界的标注图（目标 300+ 张）
2. 增加针对边界的数据增强（弹性变形、随机擦除边界区域）
3. 调整 `boundary_dilate_px` 到 5-7 px，或改用 CRF 后处理

### 通过门槛后再上船
- 先离线回放
- 再低速、短航段实船测试
- 最后逐步扩大航段和速度

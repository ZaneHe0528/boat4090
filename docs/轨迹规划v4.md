# 轨迹规划 v4：航向角 Bug 修复 & 透视修正优化

## 背景

v3 版本中发现了一个严重问题：**航向角在某些路径点达到 ±90°**，这完全不合理。

经过深入分析，发现了 3 个 Bug：

1. **相机投影公式符号错误** — 图像底部（近处地面）被错误判定为"不着地"
2. **河宽比例法用水平尺度算垂直距离** — 导致前向距离严重低估
3. **同行像素深度相同** — 导致路径末端航向角 = ±90°

本文档记录修复过程和改进方案。

---

## 问题分析

### 问题现象

运行 v3 代码的输出：

```
  [single] 11_frame_000048.jpg  heading= +50.1°  steer=+30.0°  ...
   idx   x_px  y_px   heading
  ----  ----- -----   --------
     0    330   356    +71.57°
    11    343   350    +56.31°
    ...
   165    428   149    -90.00°  ← 不合理的 -90°
   172    419   149    -90.00°  ← 同一行的两个点
```

路径末端出现了 -90° 的航向角，这表示"沿着 Y 轴方向走"，明显错误。

### Bug 1：`IMX390Projector.pixel_to_ground` 公式错误

**原始代码（v3）：**

```python
def pixel_to_ground(self, u: float, v: float) -> Optional[Tuple[float, float]]:
    dx = (u - self.cx) / self.fx
    dy = (v - self.cy) / self.fy
    
    # 错误的公式
    d_Z = dy * self.cos_p - self.sin_p
    if d_Z >= -1e-6:
        return None  # 图像底部的像素被判定为不着地！
    
    t = self.camera_height / (-d_Z)
    x_m = t * (dy * self.sin_p + self.cos_p)
    y_m = t * dx
    return (x_m, y_m)
```

**问题：**

对于图像底部的像素（v > cy，dy > 0）：
```
d_Z = 0.627 * cos(10°) - sin(10°)
    = 0.627 * 0.985 - 0.174
    = +0.443 > 0
```

`d_Z >= -1e-6` 成立，函数返回 `None`。**但这正好是最近处的地面像素，应该正常投影！**

**根本原因：** 旋转公式的方向反了。

**推导（正确）：**

相机坐标系：X（右）、Y（下）、Z（前）  
世界坐标系：X（右）、Y（前）、Z（上）

相机俯仰角 θ > 0（向下倾斜）后的旋转矩阵：

```
d_Y_w = cos(θ) - dy·sin(θ)   ← 前向分量
d_Z_w = -(sin(θ) + dy·cos(θ))  ← 竖直分量（负=向下）
```

地面交点（Z=0）：

```
0 = H - t·(sin(θ) + dy·cos(θ))
t = H / (sin(θ) + dy·cos(θ))
```

对图像底部 (dy=0.627)：
```
denom = sin(10°) + 0.627·cos(10°)
      = 0.174 + 0.617
      = +0.791  ✓ 正常
```

**修复代码（v4）：**

```python
def pixel_to_ground(self, u: float, v: float) -> Optional[Tuple[float, float]]:
    dx = (u - self.cx) / self.fx
    dy = (v - self.cy) / self.fy
    
    # 正确的公式
    denom = self.sin_p + dy * self.cos_p
    if denom < 1e-6:
        return None  # 只有地平线以上的像素返回 None
    
    t = self.camera_height / denom
    x_m = t * (self.cos_p - dy * self.sin_p)    # X = 正前方
    y_m = t * dx                                  # Y = 横向（+右 -左）
    return (x_m, y_m)
```

### Bug 2：河宽比例法用错尺度

**v3 中的问题代码：**

```python
cumulative_fwd = 0.0
for i, (px, py) in enumerate(path_pts):
    if i > 0:
        dx_px = px - path_pts[i - 1][0]      # 水平像素差
        dy_px = py - path_pts[i - 1][1]      # 竖直像素差
        # ...
        cumulative_fwd += abs(dy_px) * avg_scale  # ← 错误！
        # 用水平方向的尺度(river_width_m / width_px) 乘以竖直距离
```

**问题：** 由于透视投影，1像素水平移动代表的横向距离远小于1像素竖直移动代表的前向距离。混用导致前向距离严重低估，heading 偏大。

**修复方案（v4）：**

河宽后备模式中创建临时 IMX390 投影器用于航向角计算：

```python
# 用默认相机参数创建临时投影器，仅用于航向角的地面坐标计算
_heading_proj = IMX390Projector(camera_height=0.5, camera_pitch_deg=10.0, hfov_deg=120.0)
_heading_proj.update_resolution(w, h)

for i, (px, py) in enumerate(path_pts):
    # 地面坐标（透视修正后），仅用于航向角计算
    gnd = _heading_proj.pixel_to_ground(float(px), float(py))
    if gnd is not None:
        _gx, _gy = gnd
    elif i > 0:
        # 超出地平线：从上一个点的地面坐标逐点外推
        prev_entry = mileage_list[i - 1]
        dpx = px - path_pts[i - 1][0]
        dpy = py - path_pts[i - 1][1]
        ext_scale = river_width_m / max(rw_px, 10)
        seg = math.sqrt(dpx * dpx + dpy * dpy)
        _gx = prev_entry['_gx'] + max(seg, 1.0) * ext_scale * 3.0
        _gy = prev_entry['_gy'] + dpx * ext_scale
    else:
        _gx, _gy = 0.0, y_m
    
    mileage_list.append({
        ...
        '_gx': _gx, '_gy': _gy,  # 用于后续 _fill_heading
    })
```

### Bug 3：同行像素深度相同导致 heading = ±90°

**现象：** 路径末端多个点在 y_px=149（图像高度 h=360 时接近地平线），所有这些点的前向地面坐标 `_gx` 相同。

```
idx=165, y_px=149: _gx=1.393, _gy=1.233, heading=-90°
idx=172, y_px=149: _gx=1.496, _gy=1.130, heading=-90°
           ← 同一行，虽然 _gx 略有不同（外推），但前向步长太小
```

**修复方案：自适应滑窗法**

```python
def _fill_heading(mileage_list: List[Dict[str, Any]], window: int = 5) -> None:
    """
    基于地面坐标 (_gx=前向, _gy=横向) 为每个点计算航向角 heading_deg。
    自适应滑窗：先看前方 window 点，若前向步长过小则扩大窗口直到覆盖足够深度差。
    """
    MIN_FWD = 0.01  # 最小前向步长(m)，低于此值扩大窗口
    
    for i in range(n):
        found = False
        for w in range(window, n, window):  # 逐步扩大窗口
            j = min(i + w, n - 1)
            k = i if j > i else max(i - w, 0)
            
            d_fwd = mileage_list[j]['_gx'] - mileage_list[k]['_gx']
            d_lat = mileage_list[j]['_gy'] - mileage_list[k]['_gy']
            
            if abs(d_fwd) >= MIN_FWD:  # 足够的前向步长
                heading = math.degrees(math.atan2(d_lat, d_fwd))
                mileage_list[i]['heading_deg'] = round(heading, 2)
                found = True
                break
        
        if not found:
            # 整条路径前向步长都太小，继承前一个点的航向
            mileage_list[i]['heading_deg'] = prev_heading
```

**工作原理：**
- 小窗口（window=5）：用于路径中间部分，精度高
- 大窗口（window=15+）：自动用于路径末端同行像素，覆盖足够距离

## 技术细节

### 为什么用 IMX390 默认参数作为河宽后备方案？

1. **IMX390 是标准传感器**，h=720, w=1280 固定，参数可复用
2. **默认相机装载假设合理**：h=0.5m, pitch=10°, HFOV=120° 是典型的河道观测参数
3. **只用于航向角计算**，不影响里程的 x_m（仍用河宽比例法），降低风险
4. **可选升级**：用户提供 `--camera-height` 后切换精确模式，向后完全兼容

### 自适应滑窗为什么有效？

- **MIN_FWD = 0.01m**：设置得足够小（约等于 1 像素乘以透视尺度）
- **逐步扩大**：w=5 → 10 → 15 → ... 直到覆盖足够前向距离
- **计算成本低**：路径长度最多 100+ 点，通常只需 2-3 次迭代

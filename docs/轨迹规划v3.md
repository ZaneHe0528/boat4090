# 轨迹规划 v3：离线模型推理测试 — 航向角 + 里程坐标

## 背景

在没有接入摄像头的情况下，使用 SegFormer 模型对已有图片做离线推理，验证完整的轨迹规划管线，输出：

- **航向角**（heading_deg）：路径在预瞄点的方向，0°=正前方，+右偏，-左偏
- **里程 x, y 坐标**：路径上每个点的世界坐标（米）
- **到达时间**：基于固定船速 0.4 m/s 的预估

---

## 代码改动摘要

### 1. 新增航向角计算

在 `_compute_path_mileage` 中增加了 `_fill_heading` 函数，对路径上每个点计算航向角：

```
航向角 heading_deg = atan2(Δx_m, Δy_fwd)

  0° = 正前方
 +N° = 向右偏 N 度
 -N° = 向左偏 N 度
```

计算方法：取当前点到下一个点的方向向量，`atan2(横向变化, 纵向变化)` 得到航向角。最后一个点沿用前一个点的航向。

### 2. 固定船速与到达时间

新增 `--boat-speed` 参数（默认 0.4 m/s），每个里程点增加 `time_s` 字段：

```
time_s = 累计里程 / 船速 = y_m / 0.4
```

### 3. 里程详表输出

每张图片处理后，控制台打印采样里程表（最多 15 行）：

```
   idx  x_px  y_px    x(m)    y(m)   heading   time(s)
  ----  ----- -----  ------- -------  --------  -------
     0    320   479   +0.000   0.000     +2.30°     0.00
    10    318   460   -0.014   0.152     +2.50°     0.38
    20    315   440   -0.035   0.305     +3.10°     0.76
    ...
```

### 4. CSV 自动保存

当指定 `--output` 目录时，每张图片的完整里程数据保存为 CSV：

```
vis_pilot/frame_001_mileage.csv
vis_pilot/frame_002_mileage.csv
...
```

CSV 格式：`idx,x_px,y_px,x_m,y_m,heading_deg,time_s`

## 坐标系定义

```
         X（正前方）
            ↑
            │
  ──────────┼──────────  Y（横向）
          -Y│+Y
     (左偏) │ (右偏)
            │
         船的位置（原点）
```

| 轴 | 方向 | 字段 |
|----|------|------|
| **X** | 正前方（前进方向） | `x_m`（累计路径里程）|
| **Y** | 横向（+右 -左） | `y_m`（横向偏移） |
| **heading** | 从 X轴正方向起算 | 0°=正前，+右偏，-左偏 |

## 航向角与舵角的关系

```
航向角 (heading_deg)           舵角 (steering_deg)
  ↓                              ↓
路径在预瞄点的                  Pure Pursuit 计算的
方向角度（从X轴量）             转向修正角度

  heading 描述的是               steering 描述的是
  「路径朝哪个方向走」           「船需要往哪边打舵」

  一般情况:
  heading > 0（路径偏右，+Y）→ steering > 0（右转跟上）
  heading < 0（路径偏左，-Y）→ steering < 0（左转跟上）

  幅度上 |steering| ≥ |heading|，因为 Pure Pursuit
  需要额外的修正量来追上偏离的路径
```

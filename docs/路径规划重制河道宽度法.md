# 警戒线中线导航改进方案

## 背景

已有 `Boundary_identification.py` 能从 SegFormer 分割结果中提取左右两侧警戒线折线。目标：以两线中线作为小船航行路径，并改进该方案的工程可靠性。

---

## 改进项 1：基于已知宽度的像素/米比例估算 ⭐⭐⭐

### 原理
在透视投影下，对**直线等宽航道**，逐行取左右边界 x 均值得到的中线在几何上是正确的（透视不破坏左右对称性）。因此**不需要 IPM 标定**即可得到正确中线。

已知两侧警戒带间距（航道宽度）为 **2.5 米**，可以利用这个先验知识做像素↔物理尺度的转换。

### 方案
每行 y 处，左右边界的像素间距为 $W_{px}(y)$，则该行的像素/米比例为：

$$\text{scale}(y) = \frac{W_{px}(y)}{2.5}\ [\text{px/m}]$$

**核心用途：**

1. **横向偏差物理量化**：中线偏离图像中心的像素差 → 实际横向偏差（米）
   ```python
   CHANNEL_WIDTH_M = 2.5
   scale_ppm = width_px / CHANNEL_WIDTH_M  # 像素/米
   cte_m = (x_center - img_w / 2) / scale_ppm  # 横向误差（米）
   ```

2. **检测异常帧过滤**：某行测量宽度偏离近端均值过大（>40%）→ 跳过该行
   ```python
   bottom_rows = [y for y in width_dict if y > img_h * 0.7]
   mean_scale = np.mean([width_dict[y] for y in bottom_rows]) / CHANNEL_WIDTH_M
   valid_rows = [y for y in overlap_ys
                 if abs(width_dict[y] / CHANNEL_WIDTH_M - mean_scale) / mean_scale < 0.4]
   ```

3. **单侧边界缺失时重建对侧**：
   ```python
   # 已知左边界 x_L(y)，用近端比例估算右边界
   x_R_virtual = x_L + scale_bottom * CHANNEL_WIDTH_M
   x_center = (x_L + x_R_virtual) / 2
   ```

### 局限性
- 航道有较大弯曲时，同一图像行左右边界深度不同，宽度假设偏差增大
- 警戒带实际间距不均匀时需现场测量修正

## 改进项 2：横向偏差时序滤波（EMA / Kalman）⭐⭐⭐

### 问题
分割模型逐帧输出有微小抖动，导致中线横向位置抖动 → 舵角频繁振荡。

### 方案
对**横向偏差（cross-track error, CTE）** 做指数滑动平均（EMA）：

```python
alpha = 0.3  # 越小越平滑，越大越响应快
cte_filtered = alpha * cte_raw + (1 - alpha) * cte_prev
```

或升级为 Kalman 滤波（状态：[cte, cte_velocity]），在观测噪声较大时效果更优。

### 在哪里加
在 `realtime_pilot.py` 的帧循环中，拿到 `center_pts` 后、传给 Pure Pursuit 之前，对图像底部中心点的 x 偏差做滤波。`trajectory_planner.py` 中的 `path_history` 机制可扩展为此用途。

---

## 改进项 3：近端 ROI 截断⭐⭐

### 问题
图像顶部（远端）边界检测可靠性低（透视压缩、波光干扰），将其纳入中线计算引入噪声。

### 方案
只使用图像下方 50%（可调）的行做中线计算：

```python
ROI_TOP_RATIO = 0.5   # 只用 y > img_h * 0.5 的行
effective_rows = [y for y in overlap_ys if y > img_h * ROI_TOP_RATIO]
```

`plan_path_from_bottom` 已有类似逻辑，可直接复用。

---


## 改进项 4：边界样条拟合（可选）⭐

### 问题
逐行扫描的折线点在局部仍有锯齿。

### 方案
用 `scipy.interpolate.splprep` 对左右边界折线做样条拟合后再取中线。`path_processor.py` 中已有 `path_smoothing_factor` 参数和样条插值代码，对边界线复用即可。

---

## 推荐实施路线

```
阶段 1（1-2天）：近端 ROI 截断 + EMA 时序滤波 + 已知宽度比例转换
  → 最低成本，立刻减少抖动，提供物理尺度

阶段 2（1天）：集成 Boundary_identification.py 的警戒线识别方法
  → 替换 realtime_pilot.py 中的边界检测，统一识别流程

阶段 3（1天）：航道宽度监控 + 异常帧过滤
  → 安全裕度感知，驱动速度策略
```

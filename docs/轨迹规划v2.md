# 轨迹规划 v2：IMX390 相机投影模型优化

## 概述

本文档详细说明了如何基于 **Sony IMX390-GMSL2** 相机的硬件参数和安装姿态，构建精确的透视逆投影模型，将图像像素坐标转换为真实世界坐标（米）。

**关键改进**：从线性近似（河宽比例法）升级到非线性精确模型，彻底解决透视畸变导致的远距离失真问题。

---

## 问题诊断

### 之前的缺陷：河宽比例法

原始方法将每行图像当作**独立的线性缩放**处理：

```
比例因子 scale(row) = 2.5m / river_width_px(row)
x_m = (x_px - img_cx) × scale(row)
```

**问题**：
1. **透视非线性忽视**：图像顶部 1 像素对应的真实距离可能是底部的 10 倍以上
2. **远距离失真大**：预瞄点距离越远，误差越严重
3. **依赖 mask 质量**：需要准确测量每行河道宽度，mask 噪声或边界检测失败时失效
4. **纵向里程不精确**：相邻两行的比例平均化处理不能正确反映透视变换

### 根本原因：忽视相机几何

相机成像遵循**针孔相机模型**（Pinhole Camera Model），满足**透视投影**的数学约束：

```
3D 世界点 → 2D 图像像素 之间存在严格的线性关系
（以齐次坐标表示）
```

当反过来（像素→世界）时，需要用**逆投影**加上安装姿态信息。

### 焦距计算

基于传感器尺寸和镜头 HFOV，可计算内参焦距（以像素为单位）：

```
焦距 (pixel) = (分辨率宽度) / (2 × tan(HFOV/2))

H120 示例:
  fx = 1920 / (2 × tan(120.6°/2))
     = 1920 / (2 × tan(60.3°))
     = 1920 / 3.467
     ≈ 554.3 px
```

## 针孔相机投影模型

### 几何直观

```
                    相机镜头
                   ╱ │ ╲
                 ╱   │h  ╲        h = camera_height
               ╱     │     ╲      θ = camera_pitch
             ╱ θ     │       ╲
    ────────╱────────┼────────╲──── 水面 (Z=0)
          近处     正下方     远处
        (像素大)            (像素小)
         
          ← 向上看 (θ<0)    向下看 (θ>0) →
```

### 数学推导

#### 第一步：像素→相机归一化坐标

```
已知:
  cx = 分辨率宽度 / 2        # 图像中心 x
  cy = 分辨率高度 / 2        # 图像中心 y
  fx = 焦距（像素）
  fy = 焦距（像素，方形像素时 fx=fy）

对于像素点 (u, v):
  dx = (u - cx) / fx        # 相机坐标系：向右
  dy = (v - cy) / fy        # 相机坐标系：向下
  dz = 1                     # 相机坐标系：向前
```

**物理意义**：
- 如果 u < cx（图像左侧），则 dx < 0（相机看向左）
- 如果 v < cy（图像上方），则 dy < 0（相机看向上）

#### 第二步：相机坐标系→世界坐标系（旋转）

相机通过**俯仰角 θ** 向下倾斜，旋转矩阵为：

```
     [1      0       0  ]       [1  0  0]
R = [0   cos(θ)  sin(θ)] =  [0  c  s]    其中 c=cos(θ), s=sin(θ)
     [0  -sin(θ)  cos(θ)]       [0 -s  c]
```

应用旋转后，世界坐标系方向向量为：

```
d_X = 1 × dx + 0 × dy + 0 × 1 = dx
d_Y = 0 × dx + cos(θ) × dy + sin(θ) × 1 = dy·cos(θ) + sin(θ)
d_Z = 0 × dx - sin(θ) × dy + cos(θ) × 1 = -dy·sin(θ) + cos(θ)
```

简化为：

```
d_X = dx
d_Y = dy·cos(θ) + sin(θ)      ← 前方方向分量
d_Z = cos(θ) - dy·sin(θ)      ← 竖直方向分量
```

#### 第三步：与水面（Z=0）相交

相机位于 (0, 0, h)，沿方向 (d_X, d_Y, d_Z) 发射光线：

```
点的参数方程:  P(t) = (0, 0, h) + t·(d_X, d_Y, d_Z)
             P(t) = (t·d_X, t·d_Y, h + t·d_Z)

与水面 Z=0 相交:
  h + t·d_Z = 0
  t = -h / d_Z = h / (dy·sin(θ) - cos(θ))
```

**条件**：t > 0 时有有效交点，即 dy·sin(θ) > cos(θ)。

#### 第四步：计算世界坐标

```
x_m = t · d_X = t · dx
y_m = t · d_Y = t · (dy·cos(θ) + sin(θ))
```

代入 t：

```
x_m = (h / (dy·sin(θ) - cos(θ))) · dx

y_m = (h / (dy·sin(θ) - cos(θ))) · (dy·cos(θ) + sin(θ))
```

### 代码实现：IMX390Projector 类

```python
class IMX390Projector:
    """
    基于 IMX390-GMSL2 传感器参数的针孔相机地面投影器。
    
    参数:
      camera_height (float):  相机离水面高度（米）
      camera_pitch_deg (float): 相机俯仰角（度，>0 向下看）
      hfov_deg (float):       水平视场角（度）
    """
    
    # IMX390 传感器固定参数
    SENSOR_W_MM = 5.81
    SENSOR_H_MM = 3.30
    NATIVE_W = 1920
    NATIVE_H = 1080
    
    def __init__(self, camera_height=0.5, camera_pitch_deg=10.0, hfov_deg=120.0):
        self.camera_height = camera_height
        self.pitch = math.radians(abs(camera_pitch_deg))
        self.hfov_deg = hfov_deg
        
        # 计算焦距（以本地分辨率为单位）
        self.native_fx = self.NATIVE_W / (2.0 * math.tan(math.radians(hfov_deg / 2.0)))
        self.native_fy = self.native_fx
        
        self._img_w = self.NATIVE_W
        self._img_h = self.NATIVE_H
        self._update_intrinsics()
    
    def _update_intrinsics(self):
        """更新内参以适应实际分辨率（如果与原生不同）"""
        sx = self._img_w / self.NATIVE_W
        sy = self._img_h / self.NATIVE_H
        self.fx = self.native_fx * sx
        self.fy = self.native_fy * sy
        self.cx = self._img_w / 2.0
        self.cy = self._img_h / 2.0
        self.sin_p = math.sin(self.pitch)
        self.cos_p = math.cos(self.pitch)
    
    def pixel_to_ground(self, u, v):
        """
        像素 (u, v) → 水面坐标 (x_m, y_m)
        
        返回:
          (x_m, y_m) 如果像素在水面上方
          None 如果像素投影不到水面
        """
        dx = (u - self.cx) / self.fx
        dy = (v - self.cy) / self.fy
        
        d_Z = dy * self.cos_p - self.sin_p
        if d_Z >= -1e-6:  # 像素在地平线或下方，无法投影
            return None
        
        t = self.camera_height / (-d_Z)
        x_m = t * dx
        y_m = t * (dy * self.sin_p + self.cos_p)
        
        return (x_m, y_m)
```

该函数现在支持两种模式：

#### 模式 1：相机投影（当 projector != None）

```python
def _compute_path_mileage(path_pts, mask, river_width_m=2.5, projector=None):
    if projector is not None:
        # 精确模式
        mileage_list = []
        cumulative_m = 0.0
        prev_ground = None
        
        for i, (px, py) in enumerate(path_pts):
            gnd = projector.pixel_to_ground(float(px), float(py))
            if gnd is None:
                gnd = prev_ground if prev_ground else (0.0, 0.0)
            
            if i > 0 and prev_ground:
                seg_m = math.sqrt((gnd[0]-prev_ground[0])**2 + (gnd[1]-prev_ground[1])**2)
                cumulative_m += seg_m
            
            mileage_list.append({
                'x_px': int(px), 'y_px': int(py),
                'x_m': round(gnd[0], 3),
                'y_m': round(cumulative_m, 3),
            })
            prev_ground = gnd
        
        return mileage_list
```

**优势**：
- 无需测量 mask 中的河道宽度
- 透视变换处理精确
- 远距离离散化少（不再逐行线性）

### 安装参数测量

#### 1. camera_height（相机高度）

用卷尺或直尺测量相机镜头中心到**静止水面**的垂直距离：

```
     相机
      ↓
    ┌─────┐
    │镜头 │
    └──┬──┘
       │ ← 垂直测量
       │
    ════════════ 水面
    
    h = 0.5m
```

**注意**：
- 水面要静止，避免波浪
- 测量点应该在相机视野范围内
- 如果相机固定在船上，每次启动前重新测量（船的吃水可能变化）

#### 2. camera_pitch（俯仰角）

用手机水平仪 App 贴在相机上侧面，读取相对于水平线的角度：

```
  ┌────────────┐
  │   IMX390   │  ← 贴水平仪 App 在这里
  │            │
  └──┐      ┐──┘
     │  10° │  ← 向下 10°
     │──────┴─────── 水平线
```

**或者估算法**：
- 如果相机能看到大约 0.5m 外的水面，pitch ≈ 10°
- 如果相机能看到大约 1m 外的水面，pitch ≈ 5°

#### 3. camera_hfov（水平视场角）

查看镜头型号或数据手册：

| 镜头 | HFOV |
|------|------|
| H60  | 63.9 |
| H120 | 120.6 |
| H190 | 186 (鱼眼) |

## 常见问题

### Q1：为什么有些像素投影不到水面？

**A**：当像素在地平线或地平线下方时，投影线与水面平行或向上，无有效交点。这在俯仰角较大或看向图像上方边缘时会出现。代码中返回 `None`，此时使用近邻点的坐标。

### Q2：怎样判断安装参数是否合理？

**A**：
1. 观察最近处（图像底部）和最远处（图像顶部）的里程变化
2. 最近处应该是 0.1m ~ 0.3m，最远处应该是 2m ~ 5m
3. 如果最远处超过 10m，说明 pitch 太小或 height 太大

### Q3：能否不修改参数就改进精度？

**A**：可以结合两种模式：
- 用投影模型计算主路径（纵向 y_m）
- 用河宽比例法补充横向 x_m（如果 mask 质量好）

这样可以兼顾精度和鲁棒性。

### Q4：如果相机移动了怎么办？

**A**：需要重新测量三个参数。建议在启动脚本中打印相机信息，便于日后追溯。
